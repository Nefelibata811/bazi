## 1. 历法层 — 儒略日与公历互转

### 文件
`lib/domain/services/julian_day.dart`

### 算法来源
Meeus《Astronomical Algorithms》第 7 章。

### 公式（整数儒略日，正午 12:00 为整数日）

```
设 y, m, d 为公历年月日。

若 m ≤ 2: y = y - 1, m = m + 12

a = ⌊y / 100⌋
b = 2 - a + ⌊a / 4⌋

JD = ⌊365.25 × (y + 4716)⌋ + ⌊30.6001 × (m + 1)⌋ + d + b - 1524
```

### 浮点儒略日（精确到秒，用于天文计算）

```
JD_frac = JD_int + (h × 3600 + m × 60 + s) / 86400
```

### 反算（JD → DateTime）

按标准公式展开，从 JD 恢复公历日期、时、分、秒。

### 参考基准

| 日期 | 儒略日 | 说明 |
|------|--------|------|
| 1900-01-01 12:00 | 2415020 | 日柱基准点（甲戌日） |
| 2000-01-01 12:00 | 2451545 | J2000.0 基准点 |
| 2024-01-01 12:00 | 2460310 | 当前年份参考 |

## 2. 历法层 — 农历映射器

### 文件
`lib/infrastructure/calendar/basic_lunar_solar_mapper.dart`

### 算法来源
香港天文台公开算法，编码年表取自寿星万年历（1900-2100 年）。

### 编码格式

每一年用一个 32 位整数编码：
```
bits 0-11:  逐月大小月标记（1=大月 30 天，0=小月 29 天），正月为最低位。
bits 12-15: 闰月月份（0=无闰月）
bits 16-19: 闰月大小（1=大, 0=小）
bits 20-23: 正月初一公历月份
bits 24-28: 正月初一公历日期
```

### 农历 → 公历

1. 解析编码 → 得正月初一公历日期和当月儒略日
2. 从正月遍历到目标月，逐月累加大小月天数
3. 若该年有闰月 → 插闰月天数
4. 最终 `JD = 正月JD + 累计天数 + 日期 - 1`
5. 儒略日反算回公历

### 公历 → 农历

1. 输入公历日期 → 转儒略日
2. 在 1900-2100 年范围内逐农历年搜索
3. 每年从正月开始逐月累加天数
4. 找到 JD 所在月份 → 返回农历年月日

### 适用范围
1900-2100 年。范围外返回 null，由 LayeredCalendarConverter 降级为 approximate 或 placeholder。

## 3. 历法层 — 节气天文精算

### 文件
`lib/infrastructure/calendar/astro_solar_term_provider.dart`

### 算法来源
Meeus《Astronomical Algorithms》第 27 章，VSOP 近似公式。

### 核心公式

#### 太阳视黄经 `L_☉(JD)`

```
T = (JD - 2451545.0) / 36525          ← 儒略世纪数

L0  = 280.46646 + 36000.76983T + 0.0003032T²    ← 平黄经
M   = 357.52911 + 35999.05029T - 0.0001537T²    ← 平近点角

Mr = M × π / 180

C = (1.914602 - 0.004817T - 0.000014T²) × sin(Mr)
  + (0.019993 - 0.000101T) × sin(2Mr)
  + 0.000289 × sin(3Mr)                          ← 中心差

ω = 125.04 - 1934.136T                           ← 升交点平黄经
δ = -0.00479 × sin(ω × π/180)                    ← 章动（简化）

ε = -0.00569                                      ← 光行差

L_☉ = (L0 + C + δ + ε) mod 360
```

#### 牛顿迭代求解节气时刻

```
function solarTermJD(year, termIndex):
    target = (285 + 15 × termIndex) mod 360   ← 每个节气间隔 15°
                                              ← 小寒 285°, 大寒 300°, 立春 315°...

    JD₀ = 节气近似日期（来自 SolarTermConstants 表）

    for i = 1 to 6:
        L = L_☉(JD)
        Δ = target - L                    ← 角度差
        归一化 Δ 到 [-180, 180]

        if |Δ| < 0.00005:                 ← 约 4.3 秒精度
            break

        JD += Δ / (均值日运动速率 + 中心差一阶导数)

    return JD
```

### 精度

| 指标 | 值 |
|------|-----|
| 收敛阈值 | 0.00005°（约 4.3 秒） |
| 黄经公式 | VSOP 近似 |
| 章动模型 | 简化单主项（约 ±20"） |
| 与天文年历偏差 | 通常 ≤ 15 分钟 |
| 适用范围 | 1900-2100 年 |

### 4.3 十神判定

已知日主天干 DM，目标天干 T：

```
1. 同类五行（DM 与 T 同五行）:
   同阴阳 → 比肩
   异阴阳 → 劫财

2. 我生（DM 生 T）:
   同阴阳 → 食神
   异阴阳 → 伤官

3. 我克（DM 克 T）:
   同阴阳 → 偏财
   异阴阳 → 正财

4. 克我（T 克 DM）:
   同阴阳 → 七杀
   异阴阳 → 正官

5. 生我（T 生 DM）:
   同阴阳 → 偏印
   异阴阳 → 正印
```

### 4.4 纳音

60 甲子纳音表，两柱一组共享同一纳音。
- 例：甲子 / 乙丑 → 海中金（1-2），丙寅 / 丁卯 → 炉中火（3-4），壬戌 / 癸亥 → 大海水（59-60）

### 4.5 十二长生

阳干顺行（甲→亥长生, 乙→午长生等），阴干逆行（乙→午长生, 丁→酉长生等）。

## 5. 命理层 — 四柱推算

### 文件
`lib/infrastructure/calendar/precise_four_pillars_calculator.dart`

### 5.1 年柱

```
1. 取节气表中立春时刻（index=2）
2. 出生在立春之前 → 上一年干支
3. 出生在立春之后 → 今年干支
4. 干支序 = (年份 - 1984) mod 60
5. 1984 年为甲子年（基准点）
```

### 5.2 月柱

```
1. 遍历节气表"节"类项（index 为偶数），按时间降序排列
2. 找到出生时间所在的最近节令 → 取 termMonth → 得到月支序
3. 月干由年干和月支序推出：
   "甲己丙作首，乙庚戊为头，丙辛庚起，丁壬壬位，戊癸甲寅求"
   即月干 = (首月干序 + 月支序) % 10

月支与节令对应：
寅(0)=立春, 卯(1)=惊蛰, 辰(2)=清明, 巳(3)=立夏,
午(4)=芒种, 未(5)=小暑, 申(6)=立秋, 酉(7)=白露,
戌(8)=寒露, 亥(9)=立冬, 子(10)=大雪, 丑(11)=小寒
```

### 5.3 日柱

```
1. 出生公历 → 儒略日 JD
2. 基准点：1900-01-01 12:00 JD = 2415020，干支序号 = 10（甲戌）
3. 天数差 = JD - 2415020
4. 日干支序号 = (10 + 天数差) mod 60
5. 天干 = stems[序号 % 10]
6. 地支 = branches[序号 % 12]
```

### 5.4 时柱

```
1. 时支 = ⌊(时 + 1) / 2⌋ mod 12
2. 时干由日干和时支推出：
   "甲己日甲子时起，乙庚日丙子时起，丙辛日戊子时起，
    丁壬日庚子时起，戊癸日壬子时起"
   即时干 = (首时干序 + 时支序) % 10
```

### 5.5 四柱解译

每柱推得 stem / branch 后，调用 BaziRuleEngine：

```
Pillar(
  stem, branch,
  tenGod = engine.tenGodFor(dayMaster, stem),
  hiddenStems = engine.hiddenStemsOf(branch) → 每藏干 tenGod,
  naYin = engine.naYinOf(stem, branch),
  growthPhase = engine.growthPhaseOf(dayMaster, branch),
)
```

## 6. 命理层 — 大运流年推算

### 文件
`lib/infrastructure/calendar/real_luck_cycle_calculator.dart`

### 6.1 顺逆排法

```
阳年男 / 阴年女 → 顺排（从月柱干支往后推）
阳年女 / 阴年男 → 逆排（从月柱干支往前推）
```

### 6.2 起运岁数

```
1. 顺排：取出生之后的最近节气
2. 逆排：取出生之前的最近节气
3. 计算出生时刻与该节气的天数差 D
4. 3 天 = 1 岁
5. 起运岁数 = round(D / 3)
```

### 6.3 十步大运干支

```
for i = 1 to 10:
    大运天干 = stems[(月干序 + i) mod 10]    // 顺排
    大运地支 = branches[(月支序 + i) mod 12]
    大运十神 = engine.tenGodFor(dayMaster, 大运天干)
    起始年 = 起运年 + (i-1) × 10
    起始岁 = 起运岁 + (i-1) × 10
    duration = 10 年
```

### 6.4 流年干支

```
每步大运包含 10 年流年：
    流年干支 = 从大运干支出发，顺推 1-10 位
    流年十神 = engine.tenGodFor(dayMaster, 流年天干)
```

## 7. 命理层 — 格局判定

### 文件
`lib/infrastructure/calendar/rule_pattern_analyzer.dart`

### 算法（基于《子平真诠》月令取格法）

```
1. 取月支 → 本气藏干 → 格神天干

2. 日主见格神 → 十神关系 → 格局名
   (正官→正官格, 七杀→七杀格, 正财→正财格, ...,
    比肩→建禄格, 劫财→月刃格)

3. 透干检测：
   月干透格神 → +0.25 置信度，主格清纯
   年干透格神 → +0.10 置信度，格神远透
   时干透格神 → +0.10 置信度，格神通根
   均未透干 → 格神藏月支，取格稍弱

4. 混杂检测：
   正官+七杀共存 → 官杀混杂
   正财+偏财共存 → 财星混杂
   正印+偏印共存 → 印星混杂
   有混杂 → -0.05 置信度

5. 置信度 = 0.5 + 透干加分 - 混杂扣分，clamp[0.2, 0.95]

6. 兼格检测：
   月干透出 ≠ 格神 ≠ 日主 → 可构成兼格参考
```

## 9. 命理层 — 用神分析

### 文件
`lib/infrastructure/calendar/rule_useful_god_analyzer.dart`

### 算法

```
1. 日主旺衰三阶打分：

   月令打分（五行当令/生/克/休）:
   - 日主 = 月令 → +3（当令）
   - 日主生月令 → +1（相）
   - 日主克月令 → -1（休）
   - 月令克日主 → -2（囚）

   天干印比助力:
   - 比肩/劫财 → +1
   - 正印/偏印 → +1
   - 正官/七杀 → -1

   地支藏干印比助力:
   - 比肩/劫财 → +1
   - 正印/偏印 → +1

2. 强弱判定：
   得分 ≥ 5  → 日主偏强
   得分 2-4  → 日主中和偏旺
   得分 -1至1 → 日主中和
   得分 -3至-2 → 日主中和偏弱
   得分 ≤ -4 → 日主偏弱

3. 用神推荐：
   偏强 → 克泄耗为用（食伤财官杀）
   偏弱 → 生扶为用（印比）
   中和 → 按格局推荐用神
```
